⚡ Vite HMR 原理实现

通过实际可观察的例子，理解 import.meta.hot 的工作机制

一、HMR 工作流程

  1. 建立 WebSocket 连接
    浏览器通过 /@vite/client 与 Vite Dev Server 建立长连接 (vite-hmr 协议)。
  2. 文件变化监听
    Vite 使用 chokidar 监听项目源码文件的变化。
  3. 计算受影响模块
    通过模块依赖图（ModuleGraph）找到变化文件的所有被依赖者， 直到遇到「HMR 边界」（即调用了 import.meta.hot.accept 的模块）。
  4. 推送 update 事件
    服务端通过 ws 向客户端发送 { type: 'update', updates: [...] }。
  5. 客户端动态 import
    客户端把对应模块的 URL 加上时间戳（?t=xxx）后用动态 import 重新加载，并执行 accept 回调，完成局部更新。
  6. 无法局部更新时
    若某个模块没有 accept，会冒泡到调用方；如果一直冒泡到入口仍无人 accept， Vite 就会触发 full-reload（整页刷新）。

二、HMR 实战演示

修改下面对应的源文件并保存，观察浏览器是整页刷新还是仅更新模块：

① 自更新模块（accept 自身）

文件：src/counter.js

修改 src/counter.js 中的初始值或文案，组件会就地更新（状态可能丢失）。

② 数据模块（accept 依赖）

文件：src/data.js

修改 src/data.js 中的数组内容，列表会重渲染但保留计数器状态。

③ 样式 HMR

文件：src/widget.css

这是一个带样式的 widget 🎨
修改 src/widget.css 颜色或字体，样式会瞬间更新且 不刷新页面。

④ 无 accept（触发 full-reload）

文件：src/no-accept.js

修改 src/no-accept.js，由于没有任何模块 accept，会触发整页刷新。
